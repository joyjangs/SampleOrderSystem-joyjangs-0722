## 2. 배경 (Background)

가상의 반도체 회사 **"S-Semi"** 는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체,
대학 연구실 등의 고객에게 납품한다.

시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

최근 주문량 급증으로 다음과 같은 문제가 발생했다.

- "이 주문 처리됐나요?" — 주문 처리 여부를 추적할 수 없음
- "공정 예약을 했는데 언제 완성되는지 모르겠어요." — 생산 일정 가시성 부재
- "이미 충분한 재고가 있는데 왜 추가 공정이 돌아가고 있나요?" — 재고와 생산 현황 불일치

기존에는 엑셀과 메모장으로 주문을 관리하여 실수가 잦고, 재고·공정 현황을 한눈에 파악하기 어려웠다.
이를 해결하기 위해 체계적인 **"반도체 시료 생산주문관리 시스템"** 을 개발한다.

## 3. 목표 (Goals)

- 시료 등록/조회/검색을 통한 시료 마스터 데이터 관리
- 고객 주문 접수부터 승인/거절, 생산, 출고까지 전 과정의 상태 추적
- 재고 상황에 따른 자동 생산 배정(생산 큐, FIFO) 처리
- 담당자가 시스템 전체 현황(주문 상태별 건수, 재고 현황)을 한눈에 모니터링

- 다중 생산 라인 동시 운영에 대한 정교한 스케줄링 알고리즘 (FIFO 단일 전략만 지원)
- 인증/권한 관리, 다중 사용자 동시 접속 처리
- 결제, 배송 추적 등 물류 후속 프로세스

## 5. 역할 및 흐름 (Actors & Flow)

| 역할 | 설명 |
|---|---|
| 고객 (Customer) | 필요한 시료를 이메일 등으로 요청하는 시료 요청자 |
| 주문 담당자 (Order Manager) | 고객 요청에 맞게 주문서를 작성(주문 접수)하고 관리 |
| 생산 담당자 (Production Manager) | 개발 시료를 등록하고, 주문 수신 후 승인/거절 및 생산·출고를 처리 |

**흐름:** 고객 → (시료 요청) → 주문 담당자 → (주문서 전달) → 생산 담당자 → (승인/거절) → 주문 담당자

## 6. 시스템 개요

### 6.1 생산 라인
- 공장에서 시료 하나를 생산하는 설비 흐름을 의미
- 하나의 생산 라인은 시료를 한 번에 하나씩 생산 (단일 라인, FIFO 큐)
- 주문이 들어온 시료에 대해서만 생산 진행

### 6.2 시스템 형태
- 콘솔 기반으로 동작하며 담당자가 직접 명령(메뉴 번호 등)을 입력
- 시료를 등록하고 주문을 처리하는 것이 핵심 동작

### 6.3 전체 상태 흐름도

```
[주문 접수]                 RESERVED
                               │
                          승인 여부 판단
                    ┌──────────┴──────────┐
                  거절                   승인
                    │                     │
                REJECTED              재고 확인
              (모니터링 제외)      ┌─────────┴─────────┐
                                 재고 충분           재고 부족
                                   │                   │
                              CONFIRMED           PRODUCING
                              (출고 대기)         (생산 큐 등록)
                                   │                   │
                                   │              생산 완료 시
                                   │            PRODUCING→CONFIRMED
                                   │                   │
                                   └─────────┬─────────┘
                                          출고 처리
                                             │
                                          RELEASE
                                             │
                                        (고객 도달)
```

### 6.4 주문 상태 정의

| 상태 | 의미 |
|---|---|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 (정상 흐름 외 상태이며 **모니터링에서 제외**) |
| `PRODUCING` | 주문 승인 완료 및 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 및 출고 대기 중 |
| `RELEASE` | 출고 완료 |

## 7. 기능 요구사항 (Functional Requirements)

### 7.1 메인 메뉴
- 기능별 선택 화면(메뉴)을 출력
- 전체 시료에 대한 요약 정보 표시 (등록 시료 수, 총 재고, 전체 주문 수, 생산 라인 대기 건수 등)
- 하위 메뉴: 시료 관리 / 시료 주문 / 주문 승인·거절 / 모니터링 / 출고 처리 / 생산 라인 조회

| 메뉴 | 의미 |
|---|---|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 주문 (접수/승인/거절) | 고객 주문 접수 및 생산 라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 출고 처리 | CONFIRMED 상태 주문에 대한 출고 실행 |
| 생산 라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |

### 7.2 시료 관리

시료(Sample)는 시스템의 가장 기본이 되는 단위이다. 각 시료는 고유한 이름과 속성을 가지며,
**시스템에 등록된 시료만 주문 가능**하다.

| 기능 | 설명 |
|---|---|
| 시료 등록 | 새로운 시료를 시스템에 추가. 속성 값: 시료 ID, 이름, 평균 생산시간, 수율 |
| 시료 조회 | 등록된 모든 시료 목록 확인 (현재 재고 수량 함께 표시) |
| 시료 검색 | 이름 등 속성으로 특정 시료 검색 |

**수율(Yield)**: 정상적인 시료 수 / 총 생산 시료 수
- 예: 100개 생산 중 정상 물품 90개 → 수율 0.9

### 7.3 시료 주문 (예약)

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

- 고객이 원하는 시료와 수량을 주문
- 이 시점에서 주문 상태는 `RESERVED`

**예약 시 입력 값**
- 시료 ID
- 고객명
- 주문 수량

### 7.4 주문 승인 / 거절

- 접수된 주문(`RESERVED`) 목록을 확인
- 특정 주문에 대해 승인 또는 거절 처리

**접수된 주문 목록**
- `RESERVED` 상태의 주문 목록을 표시

**주문 승인**
- 재고 상황에 따라 2가지 방식으로 자동 처리
  - 재고 충분 → 주문을 즉시 `CONFIRMED` 상태로 전환
  - 재고 부족 → 생산 라인(생산 큐)에 자동 등록, 주문 상태를 `PRODUCING`으로 전환

**승인 시점 생산량 확정 규칙 (중요)**
- 실 생산량과 생산 시간은 **승인하는 시점의 재고만 고려하여 계산 후 확정**하며, 이후 생산 큐에서
  실제 생산이 시작될 때 재고를 다시 계산하지 않는다. (앞서 승인된 주문이 아직 생산 중이더라도,
  해당 주문이 예정한 미래 재고 증가분은 현재 승인 대상 주문의 계산에 반영하지 않는다 — 승인
  시점의 실제 재고만 사용)
- 재고가 남지만 그 재고가 **이미 앞선(먼저 승인된) 주문의 출고분으로 예정되어 있다면**, 뒤에
  승인되는 주문은 그만큼을 재고로 잡을 수 없다. 즉 승인 순서대로 재고를 소진해가며 부족분을
  계산해야 한다. (뒤 주문 입장에서 "이미 남에게 배정된 재고"는 없는 것으로 취급)

  ```
  예시1) 현재 재고 0, A시료 수율 0.5
    주문1: A 50개 주문 → 부족분 50 → 실 생산량 = ceil(50/0.5) = 100
           (주문분 50개 제외한 나머지 50개는 생산 완료 후 재고로 편입)
    주문2: 주문1 생산 중 A 50개 추가 주문 → 승인 시점 재고는 여전히 0 →
           부족분 50 → 실 생산량 = ceil(50/0.5) = 100 (주문1의 미래 생산량은 고려하지 않음)
           → 생산 큐 등록, 큐에서 나와 실제 생산 시작 시에도 재고를 다시 계산하지 않고
             승인 시점에 확정된 생산량(100개) 그대로 생산

  예시2) 현재 재고 90, A시료 수율 1.0
    주문1: A 100개 주문 → 부족분 = 100 - 90 = 10 → 실 생산량 10
    주문2: A 10개 주문 → 물리적 재고는 90개로 남아있지만, 그 90개는 이미 주문1의 출고분으로
           예정되어 있으므로 주문2 입장에서 가용 재고는 0 → 실 생산량 10
           (모니터링 화면에는 물리적 재고 90개가 그대로 표시되어야 함 — 승인 계산용 "가용
           재고"와 모니터링에 보여주는 "실제 재고"는 서로 다른 개념)
  ```

- 구현 시사점: 승인 로직은 시료의 물리적 `stock` 값 자체를 바로 차감하는 것이 아니라, 승인
  순서대로 처리하면서 "아직 배정되지 않은 가용 재고"를 추적해 계산에만 사용해야 한다. 물리적
  `stock`은 실제 생산 완료/출고 시점에만 변경한다.

**주문 거절**
- 즉시 `REJECTED` 상태로 전환

### 7.5 모니터링

담당자가 시스템 현재 상태를 한눈에 파악할 수 있도록 구성한다.

**주문량 확인**
- 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 주문 목록/건수 확인
- `REJECTED`는 유효한 주문이 아니므로 집계에서 제외

**재고량 확인**
- 시료별 현재 재고 수량 확인
- 주문 대비 재고 수량에 따른 상태 표기
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 부족
  - 고갈: 재고 수량이 0

### 7.6 생산 라인

- 생산 라인 정보(현재 생산 중인 작업, 대기열)를 표시
- 주문량 대비 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산

**계산 공식**
```
실 생산량 = ceil(부족분 / 수율)
총 생산 시간 = 평균 생산시간 × 실 생산량
```

- 위 계산은 **주문 승인 시점에 1회 확정**되며(7.4의 "승인 시점 생산량 확정 규칙" 참고), 생산
  큐에서 대기 후 실제 생산이 시작될 때 재고를 재계산하지 않고 확정된 값(`ProductionJob`에
  저장된 `shortage`/`actualQuantity`/`estimatedTime`)을 그대로 사용한다.
- 생산 완료 시 주문 상태를 `PRODUCING` → `CONFIRMED`로 변경

**생산 현황 표기**
- 현재 생산 중인 시료에 대한 정보 표시 (주문 정보, 현재까지 생산량 등 — 표기 수준은 자유롭게 결정)

**대기 주문 확인**
- 생산 라인의 대기열(생산 큐)을 이용해 대기 중인 작업 목록 출력
- 스케줄링 전략: **FIFO** (선입선출)

### 7.7 출고 처리

- 재고가 충분해진 `CONFIRMED` 상태 주문에 대해 출고 처리
- 특정 주문에 대해 출고 실행 → 주문 상태를 `RELEASE`로 전환

## 8. 데이터 모델 (초안)

### 8.1 Sample (시료)

| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 시료 고유 ID (예: `S-001`) |
| name | string | 시료명 |
| avgProductionTime | number | 평균 생산시간 (min/ea) |
| yieldRate | number | 수율 (0.0 ~ 1.0) |
| stock | number | 현재 재고 수량 |

### 8.2 Order (주문)

| 필드 | 타입 | 설명 |
|---|---|---|
| orderId | string | 주문번호 (예: `ORD-20260416-0043`) |
| sampleId | string | 주문 대상 시료 ID |
| customerName | string | 고객명 |
| quantity | number | 주문 수량 |
| status | enum | `RESERVED` \| `REJECTED` \| `PRODUCING` \| `CONFIRMED` \| `RELEASE` |
| createdAt | datetime | 주문 접수 일시 |
| updatedAt | datetime | 상태 최종 변경 일시 |

### 8.3 ProductionJob (생산 큐 작업 항목)

| 필드 | 타입 | 설명 |
|---|---|---|
| orderId | string | 연관 주문번호 |
| sampleId | string | 생산 대상 시료 ID |
| shortage | number | 부족분 (주문 승인 시점 가용 재고 기준, 7.4 참고) |
| actualQuantity | number | 실 생산량 = ceil(부족분 / 수율) |
| estimatedTime | number | 총 생산 시간 = 평균 생산시간 × 실 생산량 |
| progress | number | 진행률 |

> `shortage`/`actualQuantity`/`estimatedTime`은 **주문 승인 시점에 확정되어 저장**되며, 생산
> 큐 대기 후 실제 생산이 시작될 때 재계산하지 않는다 (7.4 "승인 시점 생산량 확정 규칙" 참고).

## 9. 비기능 요구사항 및 개발 지침

### 9.1 아키텍처
- **MVC 패턴** 적용: Model / View / Controller 패키지 구조와 역할 분리

### 9.2 데이터 영속성
- 애플리케이션을 재시작해도 데이터가 유지되어야 함 (파일, JSON, DB 등 방식은 자유 선택)

  재시작해도 그대로 유지되어야 한다 (진행 중이던 생산 큐 상태 포함)

## 10. UI 참고 (예시, 실제 구현 시 자유 결정)

명세 문서에 포함된 화면 예시는 이해를 돕기 위한 참고용이며, 실제 화면 구성은 자유롭게 결정 가능하다.

- 메인 메뉴: 시스템 현황(등록 시료 수, 총 재고, 전체 주문 수, 생산 라인 대기 건수) + 메뉴 선택
- 시료 관리: 시료 목록 테이블 (ID / 시료명 / 평균 생산시간 / 수율 / 현재 재고)
- 시료 주문: 시료 ID / 고객명 / 주문 수량 입력 → 확인 → 주문번호 및 상태(`RESERVED`) 출력
- 주문 승인/거절: `RESERVED` 목록 표시 → 번호 선택 → 재고 확인 결과에 따라 승인/거절 안내
- 모니터링: 상태별 주문 건수 + 시료별 재고 현황(잔여율, 여유/부족/고갈 상태)
- 생산 라인: 현재 처리 중인 작업(진행률, 완료 예정 시각) + FIFO 대기열 목록
- 출고 처리: `CONFIRMED` 목록 표시 → 번호 선택 → 출고 처리 완료 및 상태(`RELEASE`) 안내

## 11. 용어 정리

| 용어 | 정의 |
|---|---|
| 시료 (Sample) | 반도체 웨이퍼 공정으로 생산되는 시제품 단위 |
| 수율 (Yield) | 정상 생산품 수 / 총 생산 수 |
| 생산 큐 | 재고 부족으로 생산이 필요한 주문들의 FIFO 대기열 |
| 부족분 (Shortage) | 주문 수량 - 현재 재고 (0 이하면 부족 없음) |
| 실 생산량 | 수율을 고려해 부족분을 채우기 위해 실제로 생산해야 할 수량 = ceil(부족분 / 수율) |
